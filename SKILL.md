# My Academic Writing Style

Personal academic writing style focusing on clarity, conciseness, and avoiding AI-generated patterns.

## When to Use

Use this skill when:
- Editing academic manuscripts (especially systematic reviews, research papers)
- Applying consistent style across documents
- Removing AI-generated writing patterns
- Preparing manuscripts for journal submission
- Making AI-generated text sound like my writing

---

## Style Taxonomy Overview

| Category | Priority | Rules | Description |
|----------|----------|-------|-------------|
| **A. Word Formation** | 🔴 High | A-1, A-2 | Compound words, possessive issues |

## Category A: Word Formation (High Priority)

Apply these first - they have the biggest impact on readability.

### A-1. Compound/Hyphenated Words

**Priority: 🔴 HIGH**

Avoid artificial and unnecessary compound words, but keep hyphens when they clarify meaning:

| Avoid (unnatural compounding) | Use Instead |
|-------|-------------|
| teacher-led materials development | materials development led by teachers |
| instructor-led pedagogical activities | pedagogical activities led by instructors |
| student-facing applications | applications for students |
| AI-generated materials | materials generated with AI |
| AI-assisted task design | task design with AI |
| AI-powered corpus platform | corpus platform |
| GenAI-based materials | materials with GenAI |
| discipline-specific learning resources | learning resources for specific disciplines |
| context-specific customization | customization for specific contexts |

**Principle:** Avoid informal compound labels that pack too much meaning into one modifier. Use descriptive phrases instead.

**Exception — hyphens that clarify meaning should be kept:**
- `AI-assisted plagiarism` (hyphen clarifies that AI assists the plagiarism, not the writer)
- `well-known researcher`, `decision-making process`, `teacher-reported data`, `pre-service teachers`
- `anti-GenAI`, `pro-GenAI` → rewrite as descriptive phrases, not because of the hyphen but because the label itself is informal

**En-dash (–) for ranges and relationships:**
- `2022–2023`, `parent–child interaction` — these are correct and should not be flagged as errors.

---

### A-2. Possessive 's with Non-Human Subjects

**Priority: 🔴 HIGH**

Avoid possessive 's for non-human subjects (AI, concepts, tools):

| Avoid | Use Instead |
|-------|-------------|
| GenAI's capacity | the capacity of GenAI |
| GenAI's role | the role of GenAI |
| GenAI's impact | the impact of GenAI |
| the technology's ability | the ability of the technology |
| ChatGPT's public release | the public release of ChatGPT |
| Grammarly's 24/7 availability | the 24/7 availability of Grammarly |
| teachers' pedagogical content knowledge | the pedagogical content knowledge of teachers |
| students' perceptions | the perceptions of students |
| the field's current limitations | the current limitations of the field |
| free versions' limitations | the limitations of free versions |

**Exception:** Possessive for people is acceptable (e.g., "Smith's theory").

## Category B: Word Choice (Medium Priority)

### B-1. Filler Words and Exaggerated Adjectives

**Priority: 🟡 MEDIUM**

Remove or replace filler words and exaggerated adjectives:

| Avoid | Use Instead |
|-------|-------------|
| various dimensions | dimensions |
| diverse proficiency levels | different proficiency levels |
| diverse benefits | benefits |
| diverse contexts | contexts |
| diverse implications | implications |
| significant concerns | concerns |
| significant gap | gap |
| significant learning curve | learning curve |
| substantial teacher adaptation | teacher adaptation |
| depended substantially on | depended on |
| comprehensive | (remove) |
| extensive scholarly attention | scholarly attention |
| growing research attention | research attention |
| broader EAP field | EAP field |
| broader systematic reviews | systematic reviews |
| broader discussions | discussions |
| immediate pressure | pressure |
| immediate feedback | feedback |
| rapid grammar correction | grammar correction |
| deeper learning | learning |
| deeper investigation | investigation |
| deeper understanding | understanding |
| structured professional development | professional development |
| structured training | training |
| systematically examined | examined |
| consistently emphasized | emphasized |
| detailed guidance | guidance |
| particular emphasis | emphasis |
| particular significance | significance |
| particularly salient | salient |
| particularly regarding | regarding |
| particularly prompt dependency | prompt dependency |
| fundamentally | (remove) |
| foundational contributions | contributions |
| distinctive characteristics | characteristics |
| distinct EAP materials | EAP materials |
| overarching themes | themes |
| actionable recommendations | recommendations |
| effective prompting strategies | prompting strategies |
| current evidence | evidence |
| recent systematic reviews | systematic reviews |
| recent conceptual work | conceptual work |
| valuable contributions | contributions |
| valuable syntheses | syntheses |
| valuable insights | insights |
| highly cited | cited |
| crucial | important |
| critical | (remove or use sparingly) |
| essential | (remove or use sparingly) |
| key findings | findings |
| current research | research |

**Review these filler words and delete only when they are redundant and don't add meaning:**
```
collectively, robust, concrete, broader, greater, naturally, explicitly,
merely, full-cycle, structured, just, background
```

→ Words like `often`, `also`, and `both` are NOT automatically filler — they serve logical connection and emphasis when contextually appropriate. Only remove them when they are genuinely redundant.
→ AI tends to sprinkle these words habitually without adding meaning. Judge each instance by context, not by blanket deletion.

**Priority: 🟡 MEDIUM**

| Avoid | Use Instead |
|-------|-------------|
| via | using / through |
| post hoc | later |
| solely | only |
| merely | only |
| instance | example |
| subsequent | follow-up |
| rationale | process / reason |
| specificity | information |
| comparability | comparison |
| regimes | criteria / scheme / policy |
| relevance | significance |
| actionable | action |
| competencies | competency (singular when integrated meaning) |

### B-3. Remove Unnecessary Nominalizations

**Priority: 🟡 MEDIUM**

| Avoid | Use Instead |
|-------|-------------|
| the articulation of the gap | describing the gap |
| the identification of markers | identifying markers |
| the operationalization of criteria | specifying criteria |
| The introduction of X as a concept | X is one of the key concepts |
| The transition between paragraphs | Paragraphs... |
| the concept of critical AI literacy | critical AI literacy |
| a lack of studies focusing on | (delete and state directly) |

## Category C: Sentence Structure (Medium Priority)

### C-1. "While..." Clause Beginnings

**Priority: 🟡 MEDIUM**

Avoid starting sentences with "While..." (concessive clauses):

| Avoid | Use Instead |
|-------|-------------|
| While reviewed studies documented..., few provided | Reviewed studies documented..., but few provided |
| While reviewed studies mentioned..., no validated | Reviewed studies mentioned..., but no validated |
| While learning inequality was rarely mentioned..., differential | Learning inequality was rarely mentioned..., but differential |
| While such approaches are appropriate..., they limit | Such approaches are appropriate..., but they limit |
| While we searched two major databases..., relevant | We searched two major databases..., but relevant |
| While Hong Kong students valued..., they preferred | Hong Kong students valued..., but they preferred |
| While their corpus platform improved..., evidence | Their corpus platform improved..., but evidence |
| While ChatGPT-simplified texts improved..., some | ChatGPT-simplified texts improved..., but some |
| While mitigating | and mitigate |

**Principle:** Split into two main clauses or use "but" instead.

---

### C-2. "With ~ing" Participle Constructions

**Priority: 🟡 MEDIUM**

Replace "with + ~ing" constructions with finite clauses:

| Avoid | Use Instead |
|-------|-------------|
| with instructors maintaining | while instructors maintain |
| with marked variation | and there was marked variation |
| with different prompting expertise | who had different prompting expertise |

---

### C-3. Active Voice Over Passive

**Priority: 🟡 MEDIUM**

| Avoid | Use Instead |
|-------|-------------|
| "It is recommended that..." | "Please..." / "Researchers should..." |
| "The title could be revised" | "Revise the title" |
| "This should be considered" | "Consider this" |

---

### C-4. Avoid Editorializing/Subjective Comments

**Priority: 🟡 MEDIUM**

| Avoid | Use Instead |
|-------|-------------|
| "It is rightfully so, since..." | (remove subjective judgment) |
| "just 2 weeks" | "2 weeks" (remove "just") |
| "must-have segment" | (remove or use factual description) |

---

### C-5. Avoid "Not X but Y" Contrast Structures

**Priority: 🟡 MEDIUM**

| Avoid | Use Instead |
|-------|-------------|
| "not an optional add-on, but a must-have" | (delete entirely if redundant) |
| "not treated as X but as Y" | "treated as Y" |

**Principle:** If contrast doesn't contribute to argument, state Y directly.

---

## Category D: Punctuation & Flow (Low Priority)

### D-1. Em-dashes, En-dashes, and Colons

**Priority: 🟢 LOW**

Avoid excessive em-dashes (—) and colons when simpler punctuation works. Distinguish em-dash from en-dash (–) and hyphen (-):

| Avoid | Use Instead |
|-------|-------------|
| applications—how learners use...—rather than | applications (how learners use...) rather than |
| efficiency and personalization—benefits that... | efficiency and personalization. These benefits... |
| various roles—co-creator, assistant, quality checker—but | various roles (co-creator, assistant, quality checker) but |
| authenticity—a critical consideration | authenticity. This consideration is important |
| skills—competencies that | skills. Many |
| human review. Where | human review. Where (split into separate sentences) |

**Priority for em-dash replacement:**
1. Comma
2. Parentheses with "e.g.,"
3. Conjunctions (but, and)
4. Delete the insertion entirely
5. Split into separate sentences

**Do NOT flag en-dashes (–):**
- `2022–2023` (number ranges)
- `parent–child interaction` (relationships)
- These are correct usage and distinct from em-dashes (—).

---

### D-2. Citation Placement

**Priority: 🟢 LOW**

Prefer citations at end of sentences:

| Avoid | Use Instead |
|-------|-------------|
| human ideas (Creely, 2024, p. 162) before classroom | for ethical classroom integration (Creely, 2024, p. 162) |

## Application Instructions

### Step-by-Step Process

1. **Read the entire document** to understand context
2. **Apply fixes in priority order:**
   - **First** 🔴 High priority: Compound words, possessive 's, "with ~ing"
   - **Second** 🟡 Medium priority: Filler words, "While" clauses, vocabulary
   - **Third** 🟢 Low priority: Em-dashes, citation placement
3. **Preserve all content and numbers** - only change style
4. **Verify changes** maintain meaning and academic register
5. **Check consistency** - apply same rule throughout document

### Quick Reference: Priority Checklist

```
□ A-1: Compound words decomposed
□ A-2: Non-human possessive 's removed
□ B-1: Filler words minimized
□ C-1: "While" clauses at start removed
□ C-2: "with ~ing" constructions replaced
□ B-2: Formal vocabulary simplified
□ D-1: Em-dashes replaced
```

## Search Patterns (Regex)

Use these patterns to find violations:

```regex
# A-1: Compound/hyphenated words
\w+-led \w+|\w+-facing \w+|AI-\w+|GenAI-\w+

# A-2: Possessive 's with non-human
\b(GenAI|AI|ChatGPT|technology|system|platform|tool|model|approach|method|framework|study|research|field|literature|review)'s\b

# B-1: Filler words
\b(various|diverse|significant|substantial|comprehensive|extensive|immediate|rapid|deeper|particularly|fundamentally|overarching|crucial|essential|key)\b

# C-1: While at sentence start
^While\s

# C-2: with + ~ing
with \w+ing\b

# D-1: Em-dashes
—
```

## Example Application

**Before:**
> While reviewed studies documented various GenAI applications, few provided theoretically grounded frameworks for integrating AI-generated materials into coherent instructional sequences. The technology's ability to rapidly produce multiple task variants also supports formative assessment practices, allowing instructors to provide learners with extensive practice opportunities without proportional increases in preparation time.

**After:**
> Reviewed studies documented GenAI applications, but few provided theoretically grounded frameworks for integrating materials generated with AI into coherent instructional sequences. The ability of the technology to produce multiple task variants supports formative assessment practices, allowing instructors to provide learners with practice opportunities without proportional increases in preparation time.

**Changes made:**
- "While..." → split into main clause + "but"
- "various" → removed
- "AI-generated" → "generated with AI"
- "technology's ability" → "ability of the technology"
- "extensive" → removed

## Agent Mode (Option B)

For complex documents or when quality verification is needed, use the 3-agent pipeline instead of direct rule application.

### Mode Selection

| Scenario | Use Direct (Option A) | Use Agent Mode (Option B) |
|---|---|---|
| Quick email / short text (< 500 words) | Yes | Overkill |
| Medium text (500-1500 words) | Yes | Optional |
| Long document (1500-3000 words) | Yes | Recommended |
| Full manuscript / journal submission (> 3000 words) | OK | Better |
| Need change log / audit trail | No | Yes |
| Rule conflicts expected | No | Yes |
| Consistency critical | No | Yes |

### Pipeline

```
Document → [Detector] → Findings → [Rewriter] → Rewrite → [Validator] → Result
                ↓                                    ↓
          JSON violations                      Change log + stats
```

**1. Detector** (`_agents/academic-style-detector.md`)
- Scans the full document for all style violations
- Returns structured JSON with rule_id, severity, span, suggestion
- Supports category/severity filtering

**2. Rewriter** (`_agents/academic-style-rewriter.md`)
- Applies fixes from detector findings in priority order (S1 → S2 → S3)
- Resolves conflicts between overlapping rules
- Returns rewritten text + detailed change log + statistics
- Enforces hard invariants: numbers, citations, quotes, abbreviations never changed

**3. Validator** (`_agents/academic-style-validator.md`)
- Compares original vs. rewritten text
- Runs hard preservation checks (numbers, citations, quotes)
- Evaluates quality metrics (change rate < 30%, register maintained, flow)
- Checks consistency and style authenticity
- Returns: `accept` | `accept_with_warnings` | `revise` | `rollback`

### Pipeline Execution (Explicit Sequential Calls)

When using Agent Mode, execute the three agents in strict sequence. Each step depends on the previous output.

**Step 1: Detector**
- Load `_agents/academic-style-detector.md`
- Pass the full document text as input
- Request a complete scan for all violations
- Receive structured JSON with `violations` array and `summary`

**Step 2: Rewriter**
- Load `_agents/academic-style-rewriter.md`
- Pass:
  1. Original document text
  2. Detector findings (JSON output from Step 1)
- Request fixes applied in severity order (S1 → S2 → S3) and category order (A → B → C → D)
- Receive rewritten text + change log + statistics

**Step 3: Validator**
- Load `_agents/academic-style-validator.md`
- Pass:
  1. Original document text
  2. Rewritten text from Step 2
  3. Rewriter change log (JSON from Step 2)
- Request validation with recommendation (`accept` / `accept_with_warnings` / `revise` / `rollback`)

**Execution rules:**
- Do **not** skip steps.
- Do **not** run agents in parallel — each step consumes the previous output.
- If validator returns `revise`, return to Step 2 (Rewriter) with validator feedback.
- If validator returns `rollback`, discard the rewrite and return the original text.

### Hard Invariants (All Modes)

These apply to both direct mode and agent mode:

1. **Numbers preserved**: All numerical values, dates, percentages, statistics unchanged
2. **Citations preserved**: All (Author, Year) and Author (Year) citations remain
3. **Direct quotes preserved**: Text inside quotation marks never modified
4. **No new facts**: Do not add claims, citations, or data not in original
5. **Academic register maintained**: Do not make text casual or informal
6. **Abbreviations respected**: Once-defined abbreviations remain abbreviated

## Important Notes

- **Abstract excluded**: When applying to full manuscripts, abstract may be excluded from certain changes
- **Content preserved**: Never change data, numbers, citations, or factual content
- **Academic register maintained**: Style changes should not reduce formality or academic tone
- **Consistency is key**: Apply same rule consistently throughout document
- **Context matters**: Some rules may have exceptions based on specific context
- **Agent mode availability**: Option B requires all three agent files present in `_agents/`
