# Writing Style Guide: General Guide

## 1. 인위적이고 불필요한 합성어는 피하되, 의미를 명확히 하는 하이픈은 유지하라.

| AI 스타일 | 수정 |
|---|---|
| "dual-methodological approach" | "dual methodological approach" (인위적 합성) |
| "AI assisted plagiarism" | "AI-assisted plagiarism" (하이픈 필요) |
| "writer-disengagement" | "writer disengagement" (불필요한 합성) |
| "AI system representations" | "the representations of AI system" (풀어쓰기) |
| "auto-grading" | "automated assessment" (대체어 사용) |
| "anti-GenAI teachers" | "teachers who are negative to integrating AI" (비공식 레이블 풀기) |
| "pro-GenAI teachers" | "those who support it" (비공식 레이블 풀기) |
| "full-cycle, GenAI-integrated teacher PD" | "GenAI-integrated teacher PD" (불필요한 합성어 삭제) |

→ **비공식 합성 레이블**(anti-X, pro-X)은 서술적 표현으로 풀어쓸 것.

→ **하이픈이 필요한 경우**: 두 단어가 하나의 형용사로 작동하거나, 띄어쓰기 시 의미가 모호해질 때는 하이픈을 유지한다 (e.g., AI-assisted, well-known, decision-making, teacher-reported, pre-service).

→ **en dash (–) 사용**: 숫자 범위(2022–2023), 관계/대조(parent–child interaction), 또는 합성어 내의 결합을 나타낼 때는 en dash를 사용할 수 있다. 이는 긴 삽입구에 쓰이는 em-dash(---)와 구분된다.

## 2. 소유격 's 제한

ChatGPT's 같은 표현을 지양해라. 수정본에서는 "affordances of ChatGPT"와 같은 구조 선호 → 학술 글쓰기에서 비인간 주체(AI, 개념, 도구)의 소유격을 피하려는 스타일 규범 반영.

| 피해야 할 표현 | 권장 표현 |
|---|---|
| "ChatGPT's affordances" | "affordances of ChatGPT" |
| "AI's impact" | "impact of AI" |
| "AI's pedagogical value" | "the pedagogical value of AI" |
| "GenAI's affordances" | "affordances of GenAI tools" |
| "algorithms' tendency" | "the tendency within GenAI's algorithms" |
| "students' perceptions" | "perceptions of students" |
| "teachers' judgment" | "judgment of teachers" |
| "Teng and Yip's (2025) investigation" | "Teng and Yip (2025)" (소유격+명사화 동시 제거) |

→ 예외: 사람에 대한 소유격은 허용 (e.g., "Smith's theory")

## 3. 추상화를 경계하고 구체적으로 보다 일반적인 단어를 사용하라.

## 4. 삽입용 em-dash(—)와 콜론(:)은 과도하게 쓰지 말되, en-dash(–)는 필요한 경우 사용하라.

**em-dash(—)와 콜론은** 문장을 나누거나 comma를 사용하거나 다른 방법으로 대체할 것. 하이픈(-)이나 en-dash(–)와 혼동하지 말 것.

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "--- (em-dash) 삽입" | "," (comma로 대체) | 구두점 단순화 |
| ":" (콜론 뒤 나열) | "such as" 또는 "like" | 콜론→단어 전환 |
| "---that" | "but it" | em-dash→접속사 |
| "title---'Exposing...'---accurately" | "title, 'Exposing...', well" | em-dash→comma |
| "CK includes diverse areas--- such as SLA..." | "CK includes diverse areas, such as SLA..." | em-dash→comma |
| "genre conventions--- business communication..." | "genre conventions (e.g., business communication...)" | em-dash→괄호 |
| "a framework---one that integrates" | "a framework that integrates" | em-dash 삽입구 자체를 삭제 |
| "SQD-informed strategies--- modelling, collaboration..." | "SQD-informed strategies (e.g., modelling, collaboration...)" | em-dash→괄호+e.g. |

→ **em-dash(—) 대체 우선순위**: (1) comma, (2) 괄호 "(e.g., ...)", (3) 접속사 "but"/"and", (4) 삽입구 자체 삭제. em-dash는 강조나 삽입구를 위한 매우 긴 dash(—)이다.

→ **en-dash(–)는 허용**: 숫자 범위(2022–2023), 관계/대조(parent–child interaction), 합성어 내 결합을 나타낼 때는 en-dash(–)를 사용할 수 있다. 이는 하이픈(-)보다 길고 em-dash(—)보다 짧은 기호이다.

## 5. AI-생성 문체 탈피: 불필요한 복합 명사/합성어 최소화

원문(AI 스타일): "dual-methodological approach"
수정본: "dual methodological approach" (혹은 더 자연스러운 재구성)
→ 불필요한 하이픈/합성어 사용 자제, 더 읽기 쉬운 문장 구조 선호. 단, 의미 전달이나 가독성을 위해 필요한 하이픈(예: AI-assisted, teacher-reported, well-known, pre-service)은 유지한다.

## 6. 접속사 및 전환어 사용

| 위치 | 사용 패턴 |
|---|---|
| 문장 시작 | "So," "Also," "In addition," "Similarly," |
| 문장 중간 | "but," "and," "while," |
| 문단 연결 | "Regarding," "In relation to," "Additionally," |

**AI가 사용하는 장황한 전환구를 단순화할 것:**

| AI 스타일 (장황) | 수정 (간결) | 원리 |
|---|---|---|
| "In light of these" | "Therefore" | 전환구 단순화 |
| "In sum," | "Therefore," | 요약→인과 접속사 |
| "In terms of" | "Regarding" | 장황한 전치사구 → 단일 전치사 |
| "Together," | "Thus," | 추가적→인과적 접속사 |
| "Additionally" | "Also" / "In addition" | 격식 완화 |
| "Meanwhile," | (삭제, 문장 직접 시작) | 불필요한 전환어 삭제 |
| "Next," | (삭제, 문장 직접 시작) | 불필요한 순서어 삭제 |
| "So," (비공식) | "The study shows that" / "Therefore," | 비공식 접속사 → 정식 표현 |

## 7. 어휘 선택(Vocabulary Choice)

### (1) 과도한 형용사/부사 완화 및 filler words 삭제

| 원문 (AI 생성) | 수정 (나의 스타일) | 원리 |
|---|---|---|
| "accurately captures" | "well reflects" | 절대적 표현 → 상대적 표현 |
| "valuable" | "interesting" | 가치 판단 완화 |
| "major revisions" | "some revisions" | 강도 조절 |
| "critical citation error" | "citation error" | 과장된 수식어 제거 |
| "remarkable differences" | "notable differences" | 감정적 어조 제거 |
| "empirically rich" | (삭제) | 불필요한 수식 제거 |
| "key" | "some" | 과도한 강조 완화 |
| "central" | "important" | 격식 완화 |
| "robust" | "sound" 또는 (삭제) | 과장된 수식어 제거 |
| "concrete" | (삭제) | filler 형용사 |
| "broader" | (삭제) | 모호한 강화사 |
| "greater" | (삭제) | 불필요한 비교급 |
| "structured" | "the" 또는 (삭제) | 불필요한 서술적 형용사 |
| "rapidly" | "consistently" | 과장 → 측정 가능한 부사 |
| "timely investigation" | (삭제, 동사로 대체) | 불필요한 명사화 수식 |

**AI가 과도하게 반복하는 filler words 점검:**

```
collectively, robust, concrete, broader, greater, naturally, explicitly,
merely, full-cycle, structured, just, background
```

→ 원리: AI는 이 단어들을 문맥과 상관없이 버릇처럼 반복한다. 문장의 의미를 실질적으로 바꾸지 않는 과도한 사용을 점검하고, 불필요한 경우만 삭제한다. 논리적 연결이나 강조가 필요할 때는 자연스럽게 사용할 수 있다 (e.g., "also", "both", "often").

어휘: account → description / explanation / analysis

### (2) 구체적 동사 선호

| 원문 | 수정 | 원리 |
|---|---|---|
| "articulate" | "describe" | 학술적 → 일상적 |
| "orient readers" | "lead readers" | 추상적 → 구체적 |
| "foreground" | "show" / "highlight" / "base" | 복잡한 → 단순한 |
| "elaborate on" | "explain" | 불필요한 형식성 제거 |
| "concretely" | "explicitly" | 모호한 부사 → 명확한 부사 |
| "signal" | "show" / "indicate" | 추상적 → 구체적 |
| "captures" | "reflects" | 과장 → 중립 |
| "grounds" | "explores" | 추상적 → 구체적 |
| "conflates" | "doesn't distinguish between" / "mixes A with B" | 전문 용어 → 풀어 쓴 표현 |
| "alters" | "changes" | 격식 → 일상 |
| "shaping" | "influencing" | 결정론적 → 완화된 동사 |
| "foster" | "develop" / "enhance" | 은유적 → 직접적 |
| "frame" | "present" | 이론적 → 단순 |
| "positioned" | "perceived" | constructivist 전문용어 → 인지적 단순 동사 |
| "refine" | "develop" | 구체적 → 일반적 (문맥 의존) |
| "reshaped" | "influenced" / "reconceptualized" | 과장 → 중립 (문맥 의존) |
| "gaining" | "learning" | 격식 → 일상 |
| "experimenting with" | "trying" | 격식 → 일상 |
| "experiment with" | "explore" | 격식 → 일상 |
| "engage with" | "gain" | 구어적 phrasal verb → 단순 동사 |
| "The transition between A and B needs stronger cohesion" | "The link between..." / "Please connect A more directly and clearly to B" | "transition" 지양 |

* converge, warrant, account 등의 단어 사용 자제할 것.

### (3) 라틴어/과도한 격식어 회피

| 원문 (AI 스타일) | 수정 (나의 스타일) | 원리 |
|---|---|---|
| "via" | "using a" / "through" | 라틴어 전치사 → 일상 영어 |
| "post hoc" | "later in the Discussion" | 라틴어 학술용어 → 구체적 설명 |
| "solely" | "only" | 격식체 → 일상체 |
| "merely" | "only" | 격식체 → 일상체 |
| "instance" (for instance) | "example" (for example) | 격식 완화 |
| "subsequent" | "follow-up" | 딱딱한 표현 → 자연스러운 표현 |
| "rationale" | "process" | 문맥에 맞는 구체적 단어 |
| "specificity" | "information" | 추상 명사 → 구체 명사 |
| "comparability" | "comparison" | 불필요한 명사화 완화 |
| "regimes" | "criteria" / "scheme" / "policy" | 정치적 뉘앙스 있는 단어 회피 |
| "relevance" | "significance" | 문맥에 맞는 정확한 단어 |
| "actionable" | "action" | 불필요한 접미사 제거 |
| "competencies" | "competency" (통합적 의미일 때 단수) | 문맥에 맞는 수 일치 |

### (4) 불필요한 명사화(Nominalization) 제거

AI는 동사를 명사화해서 문장을 무겁게 만드는 경향이 있다. 이를 동사형이나 단순 구조로 전환할 것.

| AI 스타일 (명사화) | 수정 (직접적 표현) | 원리 |
|---|---|---|
| "the articulation of the gap" | "describing the gap" | 명사화 → 동명사 |
| "the identification of markers" | "identifying markers" | 명사화 → 동명사 |
| "the operationalization of criteria" | "specifying criteria" | 명사화 → 동명사 |
| "The introduction of X as a concept" | "X is one of the key concepts" | 명사화 주어 → 직접 주어 |
| "The transition between paragraphs" | "Paragraphs..." (직접 주어로) | 명사화 주어 제거 |
| "The description of the PAACS" | "For PAACS" (전치사구로 간결하게) | 명사화 주어 제거 |
| "the concept of critical AI literacy" | "critical AI literacy" | 불필요한 "the concept of" 삭제 |
| "a lack of studies focusing on" | (삭제하고 직접 진술) | 장황한 gap 서술 축약 |

**비인격 명사화 주어 → 인격 주어 전환:**

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "Strategic prompt engineering enabled them to align..." | "They relied on prompt engineering techniques... to align..." | 비인격 명사화 주어 → 인격 주어 "They" |
| "Research has developed teacher-training models..." | "Teacher-training models have been developed..." | "Research"를 주어로 쓰지 않음 |

## 8. 수동태 → 능동태

| AI 스타일 | 수정 |
|---|---|
| "It is recommended that..." | "Please..." |
| "The title could be revised" | "Revise the title" |
| "This should be considered" | "Consider this" |
| ". It was argued that X" | ": X" (콜론으로 대체) |

## 9. 긴 동격 삽입구(Long Appositive) 삭제

AI는 주어 뒤에 긴 동격 삽입구를 넣는 경향이 있다. 이를 삭제하거나 별도 문장으로 분리할 것.

| AI 스타일 | 수정 |
|---|---|
| "The topic of this study, identifying linguistic and non-linguistic clues that expose ChatGPT-assisted plagiarism in undergraduate writing through faculty perceptions and textual analysis, is interesting" | "The topic of this study is interesting because it explores AI plagiarism detection..." |

→ 원리: 주어와 동사 사이에 긴 삽입구를 넣지 않는다. 문장을 나누거나 삽입구를 삭제한다.

## 10. "While..." 종속절 시작 지양

AI는 "While it mentions..." 같은 양보절로 시작하는 것을 좋아한다. 직접적 진술을 선호할 것.

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "While it mentions..." | "It mentions...but it" | 종속절 → 두 개의 주절로 분리 |
| "While the paper notes..." | "The paper describes that..." | 종속절 제거, 직접 진술 |
| "While existing work examines..." | (삭제하고 직접 진술) | 불필요한 양보 구조 제거 |

## 11. 불필요한 인용부호(Scare Quotes) 제거

AI는 개념어에 인용부호를 과다 사용한다. 일반적 학술 용어에는 인용부호를 붙이지 말 것.

| AI 스타일 | 수정 |
|---|---|
| "deviant" (인용부호 사용) | deviant (인용부호 제거) |
| "future research" (인용부호 사용) | future research (인용부호 제거) |
| *identity (re)construction* (기울임 강조) | identity (re)construction (강조 제거) |
| "generative" nature | generative nature (인용부호 제거) |
| "breakout rooms" | breakout rooms (일반 학술용어화된 표현) |

→ 원리: 직접 인용이 아닌 한, 일반 학술 용어에 인용부호나 기울임을 사용하지 않는다.

## 12. 메타포/비유적 표현 금지

학술 글쓰기에서 비유적 표현(metaphor)을 사용하지 말 것. 직접적, 서술적 표현으로 대체한다.

| AI 스타일 (비유적) | 수정 (서술적) | 원리 |
|---|---|---|
| "rosy prospects" | "promising prospects" | 비유적 형용사 → 학술적 형용사 |
| "normalize language, thereby standardizing" | "standardize" | 비유적 이중 표현 → 직접적 단일 동사 |
| "shaping teachers' practice" | "influencing teachers' practice" | 결정론적 은유 → 중립적 동사 |
| "foster authentic learning" | "develop authentic learning" | 은유적 → 직접적 |

→ **학술 글쓰기에서 metaphor를 사용하지 말 것** (reviewer 직접 코멘트).

## 13. 편집적/주관적 코멘트(Editorializing) 삭제

AI는 글 중간에 저자의 주관적 평가나 수사적 강조를 삽입하는 경향이 있다. 학술 글쓰기에서 이를 삭제할 것.

| AI 스타일 (편집적 코멘트) | 수정 |
|---|---|
| "It is rightfully so, since school teachers face..." | "School teachers face..." (주관적 판단 삭제) |
| "It is not an optional add-on, but a must-have segment." | (통째로 삭제 — highly redundant) |
| "After just 2 weeks" | "after 2 weeks" ("just" 강조 삭제) |

→ 원리: 사실을 진술하라. 저자의 감정이나 판단("rightfully so", "must-have", "just")을 삽입하지 말 것.

## 14. 모호한 전환 문장 및 중복 문장 통째로 삭제

AI는 문단 사이에 모호한 전환 문장이나, 이미 말한 내용을 반복하는 문장을 넣는 경향이 있다. 이런 문장은 통째로 삭제한다.

**삭제 대상 유형 1: 모호한 전환 문장**

| 삭제된 문장 | 이유 |
|---|---|
| "Encouraging practices have been carried out to address teachers' pressing needs." | 구체적 내용 없는 막연한 전환 |
| "Several studies have investigated the relationship between them." | 구체적 내용 없는 막연한 주장 |
| "School norms or policies have profound implications for teachers integrating GenAI in teaching." | 다음 문장에서 이미 구체화됨 |

→ 대체 방법: 삭제 후 다음 문장을 "For example,"로 시작하거나, 바로 구체적 내용으로 진입.

**삭제 대상 유형 2: 중복 목적 진술**

| 삭제된 문장 | 이유 |
|---|---|
| "Addressing this gap, this study investigates..." (문헌 리뷰 끝에 반복) | 이미 Introduction에서 동일한 목적 진술함 |
| ", to utilize GenAI for fulfilling their pedagogical goals," (삽입구) | 앞뒤 문맥에서 이미 전달된 내용 |
| "because teachers are located within the school ecology," (부사절) | 불필요한 반복 정당화 |

→ 원리: 한 번 말한 것은 다시 말하지 않는다.

## 15. 수사적 질문형 제목 → 서술형 제목

| AI 스타일 | 수정 |
|---|---|
| "Why teacher development must add a 'critical AI literacy' practice" | "Critical AI literacy for teachers' professional development" |

→ 원리: 섹션 제목에 수사적 질문("Why...", "How...", "What makes...")을 사용하지 말고, 서술형(declarative)으로 작성한다.

## 16. "not X but Y" 대조 구문 → 직접 Y 진술

AI는 "not X but Y" 구문으로 대조를 만드는 것을 좋아한다. 불필요한 경우 "not X" 부분을 삭제하고 Y만 직접 진술한다.

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "Reflection was not treated as an afterthought but as active metacognitive inquiry" | "Reflection was treated as active metacognitive inquiry" | 불필요한 대조 삭제 |
| "It is not an optional add-on, but a must-have segment." | (통째로 삭제) | redundant |

→ 원리: 대조가 논증에 실질적으로 기여하지 않으면 Y만 진술한다.

## 17. 약어(Abbreviation) 규칙

**(1) 한번 정의된 약어는 이후 계속 약어만 사용하고, 다시 풀어 쓰지(re-expand) 않는다.**

| AI 스타일 | 수정 |
|---|---|
| "Teachers' professional development (PD)..." → 이후 다시 "professional development (PD)" | "Teachers' PD..." (이미 정의했으므로 약어만 사용) |

**(2) 본문에서 한 번만 사용되는 약어는 정의하지 않는다.**

| AI 스타일 | 수정 |
|---|---|
| "Generative Pre-trained Transformers (GPTs)" (이후 GPTs 재사용 없음) | "Generative Pre-trained Transformers" (약어 삭제) |

**(3) "Jeon and colleagues (2022)" → "Jeon et al. (2022)"**

표준 인용 약어를 사용할 것. "and colleagues"는 지양.

## 18. 인용(Citation) 위치 규칙

인용은 문장 끝에 배치하는 것을 선호한다. 문장 중간에 인용이 들어가면 가독성이 떨어진다.

| AI 스타일 | 수정 |
|---|---|
| "human ideas and experiences" (Creely, 2024, p. 162) before classroom integration. | for ethical classroom integration (Creely, 2024, p. 162). |

→ 원리: 인용을 문장 끝 마침표 직전에 배치. 문장 중간 인용은 최소화.

## 19. 학술 글쓰기에서의 Hedging(완화) 패턴

학술 논문 본문에서는 단정적 주장을 피하고 hedging을 추가한다. (Reviewer 문체와 다른 맥락)

| AI 스타일 (단정적) | 수정 (hedged) | 원리 |
|---|---|---|
| "GenAI provides powerful affordances" | "GenAI may provide powerful affordances" | may 추가 |
| "teacher agency is shaped by" | "teacher agency may be influenced by" | is→may be, shaped→influenced |
| "the projective element is responsible for" | "the projective element is linked to" | 인과→상관 |
| "it fundamentally alters" | "it fundamentally changes" | 강한 동사→중립 동사 |
| "It is compounded by associated risks" | "Therefore, it may be compounded by associated risks" | 단정→추측+접속사 |

→ 원리: 경험적 증거 없이 인과적/결정론적 주장을 하지 않는다. "may", "can", "often"으로 완화.

## 20. 문장 합치기(Sentence Merging)

짧은 두 문장이 논리적으로 연결될 때, 콜론이나 접속사로 합칠 수 있다.

| AI 스타일 (분리) | 수정 (합치기) | 원리 |
|---|---|---|
| "...traditional ways of learning (Rudolph et al., 2023). For teachers, specifically," | "...traditional ways of learning: for teachers, specifically," | 마침표+대문자→콜론+소문자 |
| "...sustained engagement. It was argued that transactional interaction..." | "...sustained engagement: transactional interaction..." | 보고절 삭제, 콜론으로 연결 |

→ 원리: 두 번째 문장이 첫 번째를 구체화/예시하면 콜론으로 합친다.

## 21. 이중 동사 구조 단순화

AI는 "should prioritize cultivating"처럼 동사를 이중으로 쌓는 경향이 있다. 하나로 줄일 것.

| AI 스타일 (이중 동사) | 수정 (단일 동사) | 원리 |
|---|---|---|
| "should prioritize cultivating" | "needs to cultivate" | 이중→단일 동사 |
| "normalize language, thereby standardizing" | "standardize" | 이중→단일 동사 |
| "was found to actively mediate" | "actively mediates" | "was found to" 제거 |

## 22. 지시대명사 및 서수사 선호

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "Those factors" | "These factors" | 원거리→근거리 지시대명사 |
| "Because of that," | "Because of this," | 원거리→근거리 |
| "Secondly," | "Second," | -ly 접미사 제거 |
| "Firstly," | "First," | -ly 접미사 제거 |

## 23. 불필요한 장황한 구절(Verbose Phrases) 삭제

| AI 스타일 | 수정 | 원리 |
|---|---|---|
| "the concept of critical AI literacy" | "critical AI literacy" | "the concept of" 삭제 |
| "addressing this gap," | (삭제) | 이미 gap 진술 후이므로 불필요 |
| "for instructional supports" | (삭제) | 문맥상 자명 |
| "and pedagogical goals" | (삭제) | 앞 문맥과 중복 |
| "since GenAI-integrated PD is still in early development" | "partially because GenAI-integrated PD is still in early development" | "since"→"partially because" (인과 명확화) |

# Writing Style Guide: A Case of Research Paper

> This guide is derived from the published paper "Enhancing EFL Pre-Service Teachers' Affordance Noticing and Utilizing with the Synthesis of Qualitative Evidence Strategies" (Computers & Education, 2022). It serves as a reference for AI systems to replicate this author's academic writing style.

---

## 1. Overall Tone and Register

### 1.1. Formality Level
- **High academic register** throughout. No colloquialisms, no contractions.
- Measured, cautious, evidence-grounded tone. Claims are always hedged or supported by citations.
- Impersonal but not cold: the writing conveys intellectual engagement with the topic.

### 1.2. Author Stance
- **Dual pronoun strategy**:
  - `"we"` for describing actions taken by the research team: *"We adopted a qualitative-dominant mixed research approach"*
  - `"the study"` / `"this study"` / `"the authors"` for framing contributions and claims: *"The study conceptualizes EFL pre-service teachers as both learners and teachers"*
- **Never** uses `"I"` even when describing the researcher's own observations.
- The passive voice is used for procedures and general truths; the active voice is used when emphasizing the study's contributions or choices.

### 1.3. Hedging Profile (Cautious Assertion)

The author **consistently hedges** claims. This is a core stylistic trait.

| Hedging Device | Examples from the Paper |
|---|---|
| Modal verbs | *"may invite them to activate"*, *"can be perceived, chosen, and used"*, *"may pose technical demands"* |
| Reporting-as-distance | *"Research notes that..."*, *"It is known that..."* |
| Potential language | *"retains potential benefits"*, *"potentially enhances"* |
| Adverbs of limitation | *"relatively superior aspects"*, *"relatively lower rates"* |
| Concessive framing | *"Although..., little is known about..."* |

**Rule**: Almost no claim is made without hedging unless it is (a) a direct quotation, (b) a widely accepted fact, or (c) a procedural description of what the researchers actually did.

---

## 2. Vocabulary and Word Choice

### 2.1. Preferred Academic Verbs

The author uses a **specific set of high-register verbs** repeatedly. AI should draw from this inventory:

**For describing what studies/literature do:**
```
address, argue, conceptualize, emphasize, explore, highlight, indicate,
insist, investigate, note, observe, report, suggest, underscore
```

**For describing what this study does:**
```
adopt, aim, conceptualize, draw on, explore, illustrate, observe, suggest
```

**For describing participant/affordance actions:**
```
activate, emerge, enact, exercise, exert, exploit, facilitate, notice,
perceive, realize, tackle, utilize
```

**For describing relationships between concepts:**
```
compensate for, complement, contribute to, correspond to, eschew,
integrate, mediate, promote, retain, serve as, support
```

### 2.2. Preferred Transition Phrases

| Function | Preferred Expressions |
|---|---|
| Adding | *"Also,"* / *"In addition to"* / *"It further..."* / *"Furthermore,"* |
| Contrasting | *"However,"* / *"Unlike..."* / *"Despite..."* / *"While..."* |
| Cause/Result | *"Thus,"* / *"Therefore,"* / *"In this regard,"* / *"Given..."* |
| Reformulation | *"In other words,"* / *"Put it another way,"* / *"that is to say,"* |
| Specification | *"In particular,"* / *"Specifically,"* / *"In this vein,"* |
| Purpose | *"For this purpose,"* / *"By doing so,"* / *"To this end,"* |
| Basis | *"Drawing on..."* / *"Based on..."* / *"Guided by..."* |

### 2.3. Phrases for Identifying Research Gaps

This is a **signature move** of the writing. The author uses these constructions to justify the study:

```
"little is known about whether..."
"there is still no empirical research on how to..."
"previous studies ... focus on only a few constructs ... not providing a unified model"
"these studies have paid attention only to..."
"still remains significant for..."
```

### 2.4. Phrases for Introducing the Study's Contribution

```
"this study suggests ... as a unified ... model"
"this study first aims to explore..."
"it further explores how..."
"the study conceptualizes ... as both ... and ..."
"the authors observe the processes by which..."
```

### 2.5. Characteristic Collocations

The author repeatedly uses specific word pairings. Maintain these:

```
"noticing and utilizing"  (not "recognizing and using")
"activate their agency"   (not "exercise their autonomy")
"enact agency"            (not "demonstrate agency")
"exert agency"            (not "show agency")
"emerging affordances"    (not "appearing affordances")
"in-depth voices"         (not "detailed opinions")
"pedagogical affordances" (not "educational affordances")
"ecological perspective"  (not "environmental perspective")
"diverse mediating factors" (not "various influencing factors")
```

## 3. Sentence Structure and Grammar Patterns

### 3.1. Dominant Sentence Architectures

The author uses **five core sentence structures** with high frequency:

#### Pattern A: Concessive Lead + Main Claim
```
Structure: Although/While [concession], [main claim] (citations).

Example:  "Although numerous studies have reported educational benefits
          of virtual environment (VE), little is known about whether
          the design-based learning project of developing VEs contributes
          to pre-service EFL teachers' noticing and utilizing of its
          pedagogical affordances."
```

#### Pattern B: Participial Phrase Opening
```
Structure: [V-ing phrase / V-ed phrase], [main clause] (citations).

Examples: "Drawing on Gibson's (1986) definition of affordance as
           possibilities for action, an epistemological view on affordance
           underscores..."

          "Given the informed benefits of incorporating CALL into EFL
           classrooms, how to prepare EFL pre-service teachers with
           technological tools still remains significant..."

          "Equipped with a virtual space editor and a coding tool,
           Gather offers the integrated development environment..."
```

#### Pattern C: Gap-Then-Contribution (However pivot)
```
Structure: [Previous literature summary]. However, [gap identification].
           Thus, this study [contribution].

Example:  "However, previous studies on EFL teacher awareness of technology
           affordances focus on only a few constructs of agency respectively,
           not providing a unified model that addresses those factors in a
           single context. Also, these studies have paid attention only to
           in-service teachers' experiences... Thus, this study suggests
           the synthesis of qualitative evidence (SQD) strategies as a
           unified pre-service teacher-training model..."
```

#### Pattern D: Colon Elaboration
```
Structure: [General statement]: [specific elaboration/list].

Examples: "communication affordances were the most mentioned item (31.2%),
           followed by engaging learning environment (30.3%),
           constructability (20.1%), and multimodal sharing (18.4%)."

          "they enacted their agency to make their classroom manageable
           by maximizing the benefits of in-class activities: eschewing
           potential hurdles in L2 communication due to technological
           limitations."
```

#### Pattern E: Former/Latter Comparison
```
Structure: [Two things introduced]. The former [description],
           whereas the latter [description].

Example:  "The former afforded pre-service teachers' enhanced technological
           awareness of how to design and customize authentic virtual
           spaces for L2 students' needs, whereas the latter supported
           student interaction with their peers and learning materials
           both during and after class."
```

### 3.2. Sentence Length and Complexity

- **Average sentence length**: Long (25-45 words is common).
- Sentences frequently contain **2-3 embedded clauses** (relative clauses, participial phrases, appositives).
- Short sentences are rare and used only for emphasis or transition: *"The project included collaborative lesson-planning and micro-teaching of their peers."*

### 3.3. Voice Distribution

| Context | Voice | Example |
|---|---|---|
| Describing what previous studies did | Active | *"Lan (2015) argued that..."* |
| Describing what this study did | Active | *"We adopted a qualitative-dominant..."* |
| Describing general truths | Passive | *"affordance perception and exploitation should be learned by teachers"* |
| Describing procedures | Passive | *"Thematic analysis was used to..."* / *"A purposive sampling method was used to..."* |
| Describing data/results | Active/Mixed | *"The findings reported..."* / *"A total of 343 coding units emerged..."* |

### 3.4. Citation Integration Style

Two primary modes:

**Integral citation** (author as grammatical subject): used when **discussing a specific author's distinct argument**:
```
"Lan (2015) argued that the design of authentic VEs would support..."
"Ciekanski et al. (2021) highlighted the usefulness of..."
"Fox and McEwan (2017) argued that this way of communication control is essential..."
```

**Non-integral citation** (parenthetical): used when **synthesizing a shared claim across multiple sources**:
```
"...has been addressed as an important issue in CALL-based teacher education
(Haines, 2015; Nami, 2022; White et al., 2021)."
```

**Rule**: When 2+ sources support the same claim, use non-integral citation. When a single source makes a unique argument being discussed in detail, use integral citation.

## 4. Paragraph Construction and Cohesion Strategies

### 4.1. General Paragraph Architecture

Most paragraphs follow a **3-layer structure**:

```
Layer 1: TOPIC SENTENCE (claim or observation)
    |
Layer 2: DEVELOPMENT (elaboration, evidence, or mechanism)
    |
Layer 3: CLOSURE (implication, connection to next idea, or citation-backed warrant)
```

### 4.2. Cohesion Devices

#### 4.2.1. Lexical Cohesion (Repetition of Key Terms)
The author **deliberately repeats key terms** rather than using synonyms. This is a conscious academic strategy, not redundancy:
- "affordance(s)" appears throughout every section
- "agency" / "teacher agency" is never replaced with "autonomy" or "independence"
- "noticing and utilizing" is the fixed pair used consistently

#### 4.2.2. Reference Chains
Demonstrative references connect sentences across the paragraph:
```
"These studies offer diverse mediating factors..."
"In this regard, ..."
"This way of authentic communication has been reported by..."
"This finding indicates that..."
"To actualize this pedagogical goal, ..."
```

#### 4.2.3. Logical Connectors Between Paragraphs
Paragraph transitions use **explicit logical markers** at the beginning of the new paragraph:

```
Paragraph 1: [establishes context]
Paragraph 2: "An ecological perspective on affordances provides..."  (new angle)
Paragraph 3: "Foregoing studies on CALL have addressed..."           (prior work)
Paragraph 4: "The SQD model has integrated..."                      (solution intro)
Paragraph 5: "Therefore, this study first aims to..."               (study purpose)
```

The chain is: Context → Theory → Prior Work (+ Gap) → Proposed Solution → Study Purpose.

### 4.3. Two Distinct Paragraph Strategies for Literature

#### Strategy 1: Synthesis Mode (Compression)
Used when **multiple sources converge on one claim**. The author weaves citations into a single flowing narrative without stopping to describe any single study in detail.

```
DIAGRAM:
  [Shared Claim] ←── (Source A; Source B; Source C)
       |
  [Elaboration of the claim] ←── (Source D; Source E)
       |
  [Implication for this study]

EXAMPLE (Introduction, Paragraph 2):
  "An ecological perspective on affordances provides a holistic approach
   to how EFL teachers' ICT knowledge develops (Haines, 2015; Liu & Chao,
   2018). Although there exist many potential affordances of ICT tools,
   only certain affordances perceived by teachers can be realized into
   language teaching (Stoffregen, 2003; van Lier, 2004). In this regard,
   'affordances are discovered through perceptual learning...'
   (van Lier, 2008, p. 597)."
```

#### Strategy 2: Thesis-Support-Example Mode (Expansion)
Used when the author needs to **build an argument with specific evidence from individual studies**. The structure is:

```
DIAGRAM:
  [Thesis Statement]
       |
  [Supporting Statement with general evidence]
       |
  [Example Case 1: Author X (year) argued/suggested...]
       |
  [Example Case 2: Author Y (year) highlighted...]
       |
  [Synthesis/Implication connecting examples to thesis]

EXAMPLE (Literature Review, Paragraph 2):
  THESIS:    "The literature emphasizes the careful design of VE to
              maximize its pedagogical affordances (Lan et al., 2018;
              Levak & Son, 2017)."
  SUPPORT:   "To support L2 learning, research conceptualizes teachers
              as ICT designers who should consider the match between its
              environmental affordances and pedagogical benefits
              (Ciekanski et al., 2021; Kuure et al., 2016)."
  EXAMPLE 1: "Lan (2015) argued that the design of authentic VEs would
              support 'self-directing learning...' She suggested four
              core functions..."
  EXAMPLE 2: "Ciekanski et al. (2021) highlighted the usefulness of
              a scenario-based approach..."
  SYNTHESIS: "These studies indicate that training future teachers for
              VE development is significant for promoting their
              technological, pedagogical knowledge..."
```

**When to use which**:
- **Synthesis Mode**: Introduction (always), Discussion (mostly), and Literature Review when establishing broad consensus
- **Thesis-Support-Example Mode**: Literature Review when building a specific theoretical argument, and Findings when interpreting data against literature

## 5. Chapter-by-Chapter Writing Guide

### 5.1. Abstract

**Structure (fixed sequence):**
```
1. Gap/Background  → "Although numerous studies..., little is known about..."
2. Theoretical Framing → "Drawing on [theory A] and [theory B],"
3. Aim Statement   → "this study first aimed to... It further investigated..."
4. Method Summary  → "We adopted [method] with [data sources]"
5. Key Findings    → "The findings reported... The findings also showed..."
6. Implications    → "The study suggests... implications are suggested."
```

**Key traits**:
- Opens with a **concessive gap sentence** (Although..., little is known...)
- Uses `"first...further..."` to signal multiple aims
- Closes with implications, not findings
- Length: ~150-180 words

### 5.2. Introduction

**Rhetorical Move Structure:**

```
MOVE 1: Establishing the Territory (Paragraph 1)
  ├── General importance of the field
  ├── Challenge/problem statement ("However,...")
  └── Significance claim ("Thus,...still remains significant")

MOVE 2: Establishing the Theoretical Framework (Paragraph 2)
  ├── Theory introduction with broad citations
  ├── Key theoretical principle (with direct quote)
  ├── Implications for the field ("Therefore,...")
  └── Agency as central concept

MOVE 3: Identifying the Gap (Paragraph 3)
  ├── What previous studies have done
  ├── What factors they identified (listed with citations)
  ├── What they failed to do ("However,...not providing a unified model")
  └── This study's proposed solution ("Thus, this study suggests...")

MOVE 4: Presenting the Solution (Paragraph 4)
  ├── Solution description with evidence
  ├── Why it works (constructivist approach, benefits)
  └── Connection to this study's exploration

MOVE 5: Stating Purpose and Design (Paragraph 5)
  ├── "Therefore, this study first aims to..."
  ├── "It further explores..."
  ├── Method preview
  └── Researcher stance and observation focus
```

**Key traits**:
- **Every claim is backed by citations** - no unsupported assertions
- **Synthesis mode only** - no individual study descriptions
- **Funnel structure**: broad field → specific gap → this study
- **Multiple citations per claim** (typically 2-4 sources per parenthetical)
- Ends with a clear purpose statement using "Therefore"

### 5.3. Literature Review

**Rhetorical Move Structure:**

```
SECTION OPENING: Theoretical Foundation
  ├── Core theory definition (with seminal citation: Gibson, 1986)
  ├── Epistemological framing
  ├── "In other words,..." reformulation for clarity
  └── "Therefore,..." implication for teacher training

MID-SECTION: Evidence Building (Thesis-Support-Example mode)
  ├── Thesis claim (with 2+ citations)
  ├── General support (with citations)
  ├── Specific Author A: "argued/suggested that..."
  ├── Specific Author B: "highlighted..."
  └── "These studies indicate that..."

SECTION CLOSING: Gap + Research Questions
  ├── "However, despite the reported importance of..."
  ├── "there is still no empirical research on..."
  ├── "Thus, this study explores..."
  ├── "By doing so, the authors aim to..."
  └── Research questions (numbered list)
```

**Key traits**:
- **Mixes both synthesis and thesis-support-example modes** within the same section
- **Uses direct quotes sparingly** but strategically (for defining key concepts)
- **Closes with explicit research questions** formatted as a numbered list
- Research questions use different interrogative structures:
  - RQ1: "What [thing] emerged from [context]?"
  - RQ2: "In what aspects did [participants] perceive [thing]?"

### 5.4. Method

**Structure:**

```
OPENING PARAGRAPH: Methodological Justification
  ├── "Drawing on [Framework A], [Framework B], and [Framework C],"
  ├── "the study adopted [method] that [rationale]"
  └── Why this approach suits the data

3.1 PARTICIPANTS AND CONTEXT
  ├── Sampling method + demographics (precise numbers)
  ├── Institutional context
  └── Course description (textbook, focus, activities)

3.2 RESEARCH DESIGN
  ├── Timeline and phases
  ├── Procedural description (chronological)
  ├── SQD strategy implementation (strategy by strategy)
  └── Platform justification (with counter-argument structure):
      "Despite [advantages of alternative], the authors selected
       [choice] for the following reasons. First,... Second,..."

3.3 DATA COLLECTION
  ├── Data sources mapped to RQs
  ├── Procedural details (hours, duration, language, translation)
  └── Survey instrument description

3.4 DATA ANALYSIS
  ├── Analysis method (with seminal citations)
  ├── Coding procedure (step by step)
  ├── Validation measures
  └── Inter-rater reliability (Cohen's kappa values)
```

**Key traits**:
- **Procedural, descriptive, factual** tone
- **Heavy use of passive voice**: "was used to", "were collected", "was calculated"
- **Precise numbers** everywhere: "(17 male, 61 female)", "(67.4 h)", "(23-37 min)"
- **Justification pattern for choices**: acknowledges alternatives, then gives reasons for actual choice using "First,...Second,..." enumeration

### 5.5. Findings

**Structure:**

```
SECTION OPENING: Overview of Findings
  ├── Section roadmap ("This section provides...")
  ├── RQ-to-section mapping
  └── Organization principle

PER-AFFORDANCE SUBSECTION (repeating unit):
  ├── Participant behavior narrative (what they did)
  ├── Quantitative anchor: "(n = 49)", "(31.2%)"
  ├── Specific participant example ("One participant noticed...")
  ├── Connection to literature ("This way of... has been reported by...")
  ├── Data extract (blockquoted participant voice)
  ├── Further analysis of the extract
  └── Transition to next sub-theme
```

**Key traits**:
- **Narrative-descriptive** style: tells the story of what participants did and discovered
- **Embeds quantitative data** naturally: "(n = 49)", "(31.2%)"
- **Data extracts** are introduced with context, then followed by interpretation
- **Each extract is linked back to prior literature** immediately after presentation
- Pattern per finding: `Observation → Participant Action → Extract → Literature Connection`

### 5.6. Discussion

**Structure:**

```
PARAGRAPH 1 (RQ1 Discussion):
  ├── Context setter ("Given that...")
  ├── Summary of this study's findings for RQ1
  ├── Comparison: "While [previous studies found X], [this study found Y]"
  ├── New contributions identified
  └── Alignment with current research ("This finding corresponds to...")

PARAGRAPH 2 (RQ2 Discussion):
  ├── "As for RQ2,..."
  ├── Method distinction: "While previous studies used..., this study adopted..."
  ├── Specific findings with numbers (Mean, SD)
  ├── Interpretation: "This finding indicates that..."
  └── Broader principle (with direct quote from literature)

PARAGRAPH 3 (Implications):
  ├── "The findings provide both academic and educational implications."
  ├── "First,..." → academic implication
  ├── "Second,..." → practical implication
  ├── "Congruent with previous studies..." alignment
  ├── "In particular,..." specificity
  └── "Finally,..." closing point
```

**Key traits**:
- **Maps findings to RQs explicitly**: "For RQ1, which addressed..."
- **Comparison with prior work** is the dominant move: "While previous studies..., this study..."
- **Identifies what is new** vs. what confirms existing literature
- **Former/latter** structure for paired comparisons
- **Implications are enumerated**: "First,...Second,...Finally,..."

### 5.7. Conclusion and Limitations

**Structure:**

```
CONCLUSION (1 paragraph):
  ├── Restate main finding: "This exploratory study illustrated how..."
  ├── Restate specific contributions (2-3 sentences)
  └── Recommendation: "the authors suggested that..."

LIMITATIONS (1 paragraph):
  ├── "As with all studies, a few limitations should be acknowledged."
  ├── Limitation 1 + remedy: "First,...Thus, further studies should..."
  ├── Limitation 2 + remedy: "Second,...To compensate,...future studies can..."
  └── Each limitation paired with a specific methodological suggestion
```

**Key traits**:
- **Brief and compressed** - no new arguments
- **Each limitation is immediately paired with a future research direction**
- Uses `"should"` and `"can"` for recommendations (not `"must"`)
- Ends on a forward-looking note

## 6. Formatting and Mechanical Conventions

### 6.1. Abbreviation Introduction
- Full term first, abbreviation in parentheses: *"virtual environment (VE)"*
- Once introduced, only the abbreviation is used thereafter
- Common pattern: introduce in Abstract, reuse freely after

### 6.2. Citation Format
- APA style: `(Author, Year)` or `Author (Year)`
- Multiple citations: alphabetical, separated by semicolons: `(Bahari, 2022; Jauregi et al., 2011; Lan, 2015)`
- Page numbers for direct quotes: `(van Lier, 2008, p. 597)`

### 6.3. Listing/Enumeration
- For sequential procedures: numbered `(1)...(2)...(3)...`
- For parallel items within a sentence: `"including [A], [B], [C], and [D]"` or `"such as [A], [B], and [C]"`
- For strategies/implications across paragraphs: `"First,...", "Second,...", "Finally,..."`

### 6.4. Direct Quotation Usage
- **Sparingly used** - only for:
  - Defining theoretical concepts: *"affordances are discovered through perceptual learning..."*
  - Highlighting a particularly well-phrased insight from another author
- Always followed by page number
- Embedded within the sentence flow, not standalone

## 7. Quick-Reference Decision Tree for AI

When generating text in this style, follow this decision process:

```
STEP 1: What section am I writing?
  │
  ├── Abstract → Use the fixed 6-part structure (Section 5.1)
  │
  ├── Introduction
  │     └── USE Synthesis Mode ONLY
  │         Every sentence must have citations
  │         Follow the 5-move funnel structure
  │         End with "Therefore, this study..."
  │
  ├── Literature Review
  │     ├── Am I establishing broad consensus?
  │     │     └── USE Synthesis Mode (compress multiple sources)
  │     ├── Am I building a specific argument?
  │     │     └── USE Thesis-Support-Example Mode
  │     │         (thesis → support → "Author (year) argued..." → synthesis)
  │     └── Am I closing the section?
  │           └── USE Gap Identification → "Thus, this study..." → RQs
  │
  ├── Method
  │     └── USE Procedural-Descriptive mode
  │         Passive voice dominant
  │         Precise numbers, justification for choices
  │
  ├── Findings
  │     └── USE Narrative-Interpretive mode
  │         Observation → Participant Action → Extract → Literature Link
  │         Embed quantitative anchors (n, %)
  │
  ├── Discussion
  │     └── USE Comparative-Evaluative mode
  │         "While previous studies..., this study..."
  │         Map to RQs, enumerate implications
  │
  └── Conclusion
        └── USE Compressed-Restatement mode
            Brief, no new claims, limitations + future directions

STEP 2: Construct each sentence
  │
  ├── Does it make a claim? → Hedge it + add citation(s)
  ├── Does it describe a procedure? → Use passive voice
  ├── Does it introduce a contrast? → Use "However," / "While..." / "Unlike..."
  ├── Does it introduce this study? → Use "Drawing on..." / "Based on..."
  └── Does it conclude a paragraph? → Use "Thus," / "Therefore," + implication

STEP 3: Check cohesion
  │
  ├── Does each paragraph start with a clear topic sentence?
  ├── Are key terms repeated (not replaced with synonyms)?
  ├── Are demonstrative references used to link sentences?
  │   ("This finding...", "These studies...", "In this regard,...")
  └── Does the paragraph end with a forward-linking closure?
```

## 8. Common Pitfalls to Avoid

| DO NOT | DO INSTEAD |
|---|---|
| Use casual transitions ("Also," at paragraph start to connect ideas weakly) | Use logical connectors that signal the relationship ("In this regard,", "Building on this,") |
| Make unhedged claims ("VEs improve teaching") | Hedge appropriately ("VEs may enhance teaching practices") |
| Replace key terms with synonyms for "variety" | Repeat the exact key terms: "affordance", "agency", "noticing and utilizing" |
| Cite a single source for a broad claim | Cluster 2-4 sources for broad claims |
| Describe individual studies in the Introduction | Synthesize; save individual study descriptions for Lit Review |
| Present findings without literature connection | Always link findings back to prior work |
| State limitations without remedies | Pair each limitation with a specific future research direction |
| Use first person singular ("I") | Use "we", "the study", "the authors", or passive constructions |
| Open a sentence with "It is important to note that" or similar filler | Go directly to the point with a substantive opening |
| Use "etc." freely | Use "such as [A], [B], and [C]" with specific examples |
